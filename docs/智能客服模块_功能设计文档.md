## 1. 模块定位与背景

### 1.1 业务痛点

跨境电商运营中，客服需要同时处理 Amazon Buyer-Seller Messages、Walmart Message Center、Shopee Chat、1688 旺旺等多个平台的客户消息：

- 多平台切换频繁，消息易遗漏
- 常见问题重复回复，效率低下
- 缺乏统一的知识库与模板支撑
- 无法量化客服工作量

### 1.2 MVP 解决目标

| 痛点 | MVP 解决方案 |
|:---|:---|
| 多渠道消息分散 | **统一收件箱** — 聚合多渠道消息，支持手动导入+模拟数据 |
| 回复效率低 | **回复模板** — 一键插入标准化回复，变量自动替换 |
| 缺乏 AI 辅助 | **AI 回复建议** — DeepSeek 分析消息意图，生成回复草稿 |

### 1.3 与现有模块的关系

```
EcomIQ-RPA 统一平台
├── 📡 RPADataHub (数据采集) ←── Playwright 兜底抓取各平台消息
├── 📊 CompetitorWatch (竞品分析)
├── 🎯 LeadScraper (客户开发)
├── 🤖 AI Assistant (AI助手)   ←── 复用 DeepSeek API 调用逻辑
└── 💬 智能客服 ServiceIQ (本模块) ←── 本次 MVP 开发
```

## 2. 核心架构风险：消息从哪里来

### 2.1 现状分析

> **这是整个模块最大的架构风险 —— 必须在设计阶段正视它。**

理想情况下，消息应通过各平台官方 API 自动同步：

| 平台 | API | 现实障碍 |
|:---|:---|:---|
| Amazon | SP-API (Selling Partner API) | 需要开发者账号、通过审核、绑定品牌 |
| Walmart | Walmart Marketplace API | 邀请制接入，门槛高 |
| Shopee | Shopee Chat API | 对第三方开放程度有限 |
| 1688 | 旺旺开放平台 | 国内平台 API 生态不完善 |

**风险**：如果 API 拉不到消息，整个"消息收件箱"就是空的。

### 2.2 三层消息获取架构（架构弹性设计）

MVP 阶段不依赖任何单一消息来源，而是设计三层获取架构：

```
消息获取层（Message Acquisition Layer）
│
├── 🥇 路径1：API 同步（理想方案，预留接口）
│   ├── Amazon SP-API → 预留 adapter 接口，权限就绪后即插即用
│   ├── Walmart API   → 预留 adapter 接口
│   └── 其他平台 API   → 统一 adapter 模式，新增平台只需实现一个接口
│
├── 🥈 路径2：RPA 模拟抓取（务实兜底，RPADataHub 已有能力）
│   ├── Playwright 脚本自动登录 Amazon Buyer-Seller Messages 页面
│   ├── 抓取消息列表 + 消息详情
│   ├── 存入 cs_messages 表
│   └── 和现有电商后台数据采集逻辑完全一致
│
└── 🥉 路径3：手动导入（始终可用，零依赖）
    ├── CSV/Excel 批量导入历史消息
    ├── 数据清洗 → 标准化入库
    └── 销售团队可自行维护
```

**面试话术**：
> "消息来源有三条路径——API 拉取、RPA 模拟抓取、手动导入。API 是最理想的方案，但在没有权限的情况下，RPADataHub 的 Playwright 采集能力可以兜底。这证明了架构的弹性和工程的务实性。"

### 2.3 MVP 阶段消息来源策略

| 阶段 | 消息来源 | 说明 |
|:---|:---|:---|
| **MVP** | 模拟数据 + 手动导入 | 用 `generate_mock_data.py` 生成测试数据，确保功能可验证 |
| **二期** | + RPA 抓取 | Playwright 脚本登录 Amazon 后台抓取真实消息 |
| **三期** | + API 同步 | 获取 API 权限后接入 |

### 5.2 核心功能点

#### 消息列表

- 多平台消息聚合展示（Amazon/Walmart/Shopee/1688），平台图标标识
- 优先级色标：紧急(🔴红色边框)、普通(🟡黄色边框)、低(🟢绿色边框)
- 消息预览：客户名称、平台、订单号、消息摘要、时间
- 分页加载：每页 20 条
- 未读标识：未读消息加粗 + 左侧红点

#### 筛选与搜索

| 筛选项 | 类型 | 说明 |
|:---|:---|:---|
| 平台 | 多选复选框 | Amazon / Walmart / Shopee / 1688 / 手动导入 |
| 状态 | 多选复选框 | 待处理 / 已回复 / 已关闭 |
| 优先级 | 多选复选框 | 紧急 / 普通 / 低 |
| 分类 | 多选复选框 | 退货退款 / 物流查询 / 产品咨询 / 投诉 / 其他 |
| 关键词 | 输入框 | 搜索客户名、订单号、消息内容 |

#### 消息详情

- **客户信息卡片**：客户名称、平台、历史沟通次数
- **AI 分析结果**（如果已分析）：意图识别、情感分析
- **关联订单信息**：自动匹配订单号 → 展示订单金额、购买日期
- **消息历史**：同一订单的对话线程
- **操作区**：
  - 回复文本框 + 模板下拉插入 + 🤖 AI 建议按钮
  - 发送 / 关闭 / 标记为垃圾

#### 批量操作

- 批量标记已读/关闭

#### 手动导入

- 支持 CSV/Excel 上传
- 字段映射引导
- 导入预览 + 确认入库

## 7. 子模块三：AI 智能回复建议

### 7.1 功能概述

AI 回复引擎是智能客服的核心，扮演"辅助参谋"角色。点击 🤖 AI 建议按钮后：

```
客户消息 → DeepSeek API → ┌─ 意图识别: 退货退款/物流查询/产品咨询/投诉/其他
                          ├─ 情感分析: 正面😊 / 中性😐 / 负面😟 / 愤怒😡
                          ├─ 关键信息提取: 订单号/ASIN/金额/问题描述
                          ├─ 模板推荐: 匹配最相关的回复模板
                          └─ 回复生成: 基于模板 + AI 润色 → 可编辑草稿
```

## 9. 数据库设计（MVP）

### 9.1 MVP 只创建 3 张表

> **约束**：`cs_knowledge`、`cs_auto_rules`、`dw_cs_daily_stats`（DW层聚合表）放到二期创建。

```sql
-- ============================================================
-- 1. 客服消息表（核心表）
-- ============================================================
CREATE TABLE cs_messages (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    platform        VARCHAR(20)   NOT NULL COMMENT '消息来源: Amazon/Walmart/Shopee/1688/manual_import',
    platform_msg_id VARCHAR(100)  DEFAULT NULL COMMENT '平台消息唯一ID（手动导入时可为空）',
    shop_name       VARCHAR(100)  DEFAULT NULL COMMENT '关联店铺',
    customer_name   VARCHAR(200)  DEFAULT NULL COMMENT '客户名称',
    customer_email  VARCHAR(200)  DEFAULT NULL,
    order_id        VARCHAR(100)  DEFAULT NULL COMMENT '关联订单号',
    asin            VARCHAR(50)   DEFAULT NULL COMMENT '关联产品ASIN',
    subject         VARCHAR(500)  DEFAULT NULL COMMENT '消息主题',
    content         TEXT          NOT NULL COMMENT '消息正文',
    priority        ENUM('urgent','normal','low') DEFAULT 'normal' COMMENT '优先级',
    status          ENUM('pending','replied','closed','spam') DEFAULT 'pending' COMMENT '处理状态',
    category        VARCHAR(50)   DEFAULT NULL COMMENT '分类: return/logistics/inquiry/complaint/other',
    sentiment       VARCHAR(30)   DEFAULT NULL COMMENT 'AI情感分析: positive/neutral/negative/angry',
    ai_intent       VARCHAR(50)   DEFAULT NULL COMMENT 'AI识别的意图',
    ai_confidence   DECIMAL(4,2)  DEFAULT NULL COMMENT 'AI置信度 0-100',
    agent_id        INT           DEFAULT NULL COMMENT '处理客服ID',
    first_reply_at  DATETIME      DEFAULT NULL COMMENT '首次回复时间',
    closed_at       DATETIME      DEFAULT NULL COMMENT '关闭时间',
    tags            JSON          DEFAULT NULL COMMENT '标签数组，如 ["urgent","vip"]',
    received_at     DATETIME      NOT NULL COMMENT '消息接收时间',
    created_at      DATETIME      DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_platform_msg (platform, platform_msg_id),
    INDEX idx_status (status),
    INDEX idx_priority (priority),
    INDEX idx_category (category),
    INDEX idx_received (received_at),
    INDEX idx_shop (shop_name),
    INDEX idx_order (order_id)
    -- 注：MVP 阶段暂不创建 FULLTEXT INDEX
    -- 数据量 >10万条后建议切换 Elasticsearch，当前阶段用 LIKE 搜索即可
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='客服消息表';

-- ============================================================
-- 2. 对话回复表
-- ============================================================
CREATE TABLE cs_conversations (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    message_id      INT           NOT NULL COMMENT '关联消息ID',
    reply_content   TEXT          NOT NULL COMMENT '回复内容',
    reply_type      ENUM('manual','template','ai_generated','auto_rule') DEFAULT 'manual' COMMENT '回复类型',
    template_id     INT           DEFAULT NULL COMMENT '使用的模板ID',
    agent_id        INT           DEFAULT NULL COMMENT '操作客服ID',
    is_ai_assisted  TINYINT(1)    DEFAULT 0 COMMENT '是否AI辅助',
    sent_status     ENUM('pending','sent','failed') DEFAULT 'sent' COMMENT '发送状态',
    created_at      DATETIME      DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (message_id) REFERENCES cs_messages(id) ON DELETE CASCADE,
    INDEX idx_message (message_id),
    INDEX idx_agent (agent_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='对话回复表';

-- ============================================================
-- 3. 回复模板表
-- ============================================================
CREATE TABLE cs_templates (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    name            VARCHAR(200)  NOT NULL COMMENT '模板名称',
    category        VARCHAR(50)   DEFAULT 'custom' COMMENT '分类',
    language        VARCHAR(20)   DEFAULT 'en' COMMENT '语言',
    platform        VARCHAR(100)  DEFAULT NULL COMMENT '适用平台,逗号分隔',
    content         TEXT          NOT NULL COMMENT '模板内容（含{{变量}}）',
    usage_count     INT           DEFAULT 0 COMMENT '使用次数',
    is_active       TINYINT(1)    DEFAULT 1 COMMENT '启用/停用',
    created_by      VARCHAR(50)   DEFAULT NULL,
    created_at      DATETIME      DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category),
    INDEX idx_active (is_active),
    INDEX idx_usage (usage_count)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='回复模板表';
```

### 9.2 全文搜索策略

| 阶段 | 方案 | 说明 |
|:---|:---|:---|
| **MVP** | `LIKE '%keyword%'` | 数据量小(<1万条)，性能完全够用 |
