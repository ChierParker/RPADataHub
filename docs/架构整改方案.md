## 1. 当前架构问题分析

### 1.1 项目根目录结构现状

```
C:\Users\JackPeesao\Desktop\EcomIQ\    ← 项目根目录
├── .env                                ← 数据库配置
├── .venv/                              ← Python 虚拟环境
├── CompetitorWatch/                    ← 竞品分析模块
├── data/                               ← 空目录
├── EcomIQ/                             ← 主应用 (与根目录同名，混淆)
├── LeadScraper/                        ← 客户开发模块
├── RPADataHub/                         ← 数据采集模块
├── 整套RPA解决方案.docx                  ← 设计文档散落
├── Admin后台改造方案.docx                ← 设计文档散落
├── UI重构文档.docx                      ← 设计文档散落
└── test.txt                            ← 临时文件
```

### 1.2 识别出的六大架构问题

#### 问题一：根目录与子模块同名混淆

```
EcomIQ/                     ← 项目根目录名
└── EcomIQ/                 ← 主应用子目录也叫 EcomIQ
    ├── app.py              ← 启动命令: python -m EcomIQ.app
    └── start.bat
```

**影响**：

- 新人看到 `EcomIQ/EcomIQ/app.py` 会困惑
- `EcomIQ` 既是 Python 包名又是目录名，`from EcomIQ.app import app` 和 `python -m EcomIQ.app` 的语义不一致
- 面试展示时目录结构不专业

#### 问题二：启动脚本碎片化，功能重叠

| 脚本 | 位置 | 功能 | 问题 |
|:---|:---|:---|:---|
| `EcomIQ/start.bat` | EcomIQ 子目录 | 启动 Web 服务 | `cd /d "%~dp0.."` 然后 `python -m EcomIQ.app` |
| `EcomIQ/start_services.bat` | EcomIQ 子目录 | 启动全栈服务 | 同样先 `cd /d %~dp0..`，启动 Web + Redis + Worker + FileWatcher |
| `RPADataHub/start_all_services.bat` | RPADataHub 子目录 | 启动 RPADataHub 服务 | 硬编码绝对路径 `C:\Users\JackPeesao\...`，且启动的是 `admin_server.py` 而非 EcomIQ 统一入口 |

**影响**：

- 三个脚本功能重叠，用户不知道该用哪个
- `RPADataHub/start_all_services.bat` 硬编码了绝对路径，换机器就失效
- `EcomIQ/start_services.bat` 和 `RPADataHub/start_all_services.bat` 启动的是**不同的应用入口**（`EcomIQ.app` vs `admin_server.py`），造成运行时不一致

#### 问题三：设计文档散落根目录

| 文件 | 位置 |
|:---|:---|
| `整套RPA解决方案.docx` | 根目录 |
| `Admin后台改造方案.docx` | 根目录 |
| `UI重构文档.docx` | 根目录 |
| `智能客服模块_功能设计文档.md` | `EcomIQ/` 子目录 |
| `API采集模块_功能设计文档.md` | `RPADataHub/` 子目录 |

**影响**：

- 设计文档与业务代码混放
- `.docx` 二进制文件不应进入 Git 仓库
- 文档分散在两个子目录，查找困难

#### 问题四：模板文件重复存储

```
EcomIQ/templates/                           ← Flask app 的 template_folder
    ├── api_collect_config.html             ← 旧占位版 (已覆盖为新版)
    ├── login.html                          ← 与 RPADataHub/templates/login.html 重复
RPADataHub/templates/                       ← 蓝图的 template_folder
    ├── api_collect_config.html             ← 功能版
    ├── login.html                          ← 与 EcomIQ/templates/login.html 重复
```

**影响**：

- 同一文件两个副本，修改时容易遗漏（之前 API 采集页面 "开发中" 就是这个问题）
- 浪费磁盘空间

#### 问题五：无关代码混入

```
RPADataHub/careercompass-main/              ← 前端项目 (React/Next.js)
    ├── package.json
    ├── tailwind.config.ts
    └── src/
```

**影响**：
-与 Python 后端混合存放，破坏项目语言一致性

- 增加了不必要的目录深度
- `RPADataHub/Skill/rpa_rag_assistant/knowledge_base/` 下的 SQLite 文件也无版本管理意义

#### 问题六：empty-data 目录与临时文件

```
data/                                       ← 空目录
test.txt                                    ← 临时测试文件
```

## 2. 整改目标与原则

### 2.1 目标

| 目标 | 描述 |
|:---|:---|
| **命名规范化** | 消除 `EcomIQ/EcomIQ` 的同名嵌套，主应用包名与项目目录名解耦 |
| **启动统一化** | 合并为一个根级启动脚本，覆盖所有使用场景 |
| **文档集中化** | 所有设计文档归入 `docs/` 目录，`.docx` 二进制文件不入 Git |
| **模板去重** | 消除两套 `templates/` 目录重复，统一模板来源 |
| **代码纯净化** | 移除无关前端项目、临时文件、空目录 |

### 2.2 原则

1. **零功能变更**：不修改任何 `.py` 文件的业务逻辑，不改变路由、API、数据库连接
2. **Python import 兼容**：所有 `from EcomIQ.app import` / `python -m EcomIQ.app` 在整改后仍正常工作
3. **安全回滚**：每步整改有明确的回滚步骤
4. **Git 友好**：使用 `git mv` 保留文件历史
5. **生产环境兼容**：整改后 `start.bat` 一键启动行为不变

## 4. 整改条目明细

### 4.1 P0 — 必须立即修复 (影响系统稳定性)

#### 条目 1：删除硬编码绝对路径的启动脚本

**问题**：`RPADataHub/start_all_services.bat` 硬编码 `C:\Users\JackPeesao\...`，换机器失效。

**操作**：

```
删除: RPADataHub/start_all_services.bat
```

**回滚**：Git revert 恢复文件。

---

#### 条目 2：合并启动脚本到根目录

**问题**：`EcomIQ/start.bat` 和 `EcomIQ/start_services.bat` 分散在子目录中。

**操作**：

```
新建: start.bat       (从 EcomIQ/start.bat 复制，修改 cd 路径)
新建: start_all.bat    (从 EcomIQ/start_services.bat 复制，修改各服务路径)
删除: EcomIQ/start.bat (保留一个兼容软链)
删除: EcomIQ/start_services.bat (保留一个兼容软链)
```

**`start.bat` 新内容**：

```bat
@echo off
chcp 65001 >nul
echo ======================================
echo   EcomIQ — 电商智能工具集
echo ======================================
echo   访问地址: http://localhost:5000
echo   默认账号: admin / RPA@admin2026
echo ======================================
cd /d "%~dp0"
if exist .venv\Scripts\activate.bat (call .venv\Scripts\activate.bat)
python -m src.main.app
pause
```

**影响分析**：

- `python -m src.main.app` 等价于之前 `python -m EcomIQ.app`，只需确保 `src/main/__init__.py` 存在
- `app.py` 中 `template_folder="templates"` 和 `static_folder="static"` 是相对路径，目录移动后不受影响
- `app.py` 第 55 行 `sys.path.insert(0, _db_module_dir)` 的 `_db_module_dir` 路径需验证，但 `get_config()` 导入使用的是绝对路径计算，不受目录移动影响

---

#### 条目 3：源代码目录重组 (核心整改)

**问题**：根目录直接放置四个模块目录，`EcomIQ/EcomIQ` 同名嵌套。

**操作**（使用 Git 保留历史）：

```bash
mkdir src
git mv EcomIQ/        src/main/
git mv RPADataHub/    src/rpa/
git mv CompetitorWatch/ src/competitor/
git mv LeadScraper/   src/leads/
```

**必须同步修改的引用**（这是唯一需要改代码的地方）：

| 文件 | 原路径 | 新路径 |
|:---|:---|:---|
| `src/main/app.py` 第 54 行 | `_db_module_dir = ... + "RPADataHub"` | `_db_module_dir = ... + "src/rpa"` |
| `src/main/app.py` 第 437-461 行 | `from RPADataHub.blueprint import...` | `from src.rpa.blueprint import...` |
| | `from CompetitorWatch.blueprint import...` | `from src.competitor.blueprint import...` |
| | `from LeadScraper.blueprint import...` | `from src.leads.blueprint import...` |
| `src/rpa/blueprint.py` 第 9 行注释 | `APIDataHub.blueprint` | 注释更新 |
| `src/rpa/admin_server.py` | `from config.settings import...` | 不受影响 (相对导入) |
| 各模块 `config/settings.py` | `dotenv` 加载路径 | 需验证 `.env` 文件查找路径 |

**`app.py` 路径修改示例**：

```python
# 修改前
_db_module_dir = os.path.join(os.path.dirname(os.path.dirname(os.path.abspath(__file__))), "RPADataHub")
# 修改后
_db_module_dir = os.path.join(os.path.dirname(os.path.dirname(os.path.abspath(__file__))), "src", "rpa")
```

## 6. 约束条件与风险控制

### 6.1 绝对不可修改的内容

| 内容 | 理由 |
|:---|:---|
| ❌ `.py` 文件的业务逻辑 | 任何函数体、路由逻辑、SQL 查询不得修改 |
| ❌ `app.run()` 的 host/port/debug 参数 | 启动行为不变 |
| ❌ 数据库连接方式 | `pymysql` + `config/settings.py` 不变 |
| ❌ 前端 HTML 的 `<script>` 逻辑 | 产品功能不变 |
| ❌ `.env` 中的配置值 | 环境变量不变 |

### 6.2 允许修改的内容（仅路径引用）

| 文件 | 可修改行 | 修改类型 |
|:---|:---|:---|
| `src/main/app.py` | 第 54 行 | `sys.path.insert` 路径字符串 |
| `src/main/app.py` | 第 437 行 | `from RPADataHub.blueprint` → `from src.rpa.blueprint` |
| `src/main/app.py` | 第 447 行 | `from CompetitorWatch.blueprint` → `from src.competitor.blueprint` |
| `src/main/app.py` | 第 455 行 | `from LeadScraper.blueprint` → `from src.leads.blueprint` |
| `src/rpa/blueprint.py` | `sys.path.insert` | 无需修改 (已在闭包内) |
