# RPA 运维手册

## 1. 常见异常处理

### 1.1 登录失败
**现象**: 采集任务报 "login failed" 或 "cookie expired"
**根因**: Cookie 过期或登录态失效，通常因为账号密码变更或平台风控触发
**处理步骤**:
1. 登录对应平台确认账号状态正常
2. 检查是否触发二次验证（2FA/验证码）
3. 重新获取 Cookie 并更新采集器配置
4. 如为批量登录失败，检查代理 IP 是否被限制
**预防**: 每周主动刷新关键账号 Cookie，配置登录态监控告警

### 1.2 数据为空
**现象**: 采集完成但返回 0 行数据
**根因**: 目标页面当日无数据产生，或查询条件不匹配
**处理步骤**:
1. 登录平台手动确认对应日期是否有数据
2. 检查业务日期参数是否正确
3. 确认是否为平台数据延迟（某些报表 T+1 才能查）
4. 如确认无数据，标记为 NO_DATA 不再重试
**预防**: 配置"数据为空"告警阈值，连续 N 天空数据触发升级

### 1.3 网络超时
**现象**: "Connection timeout" 或 "ERR_CONNECTION_RESET"
**根因**: 网络波动、代理异常、目标服务限流
**处理步骤**:
1. ping 目标域名确认网络可达
2. 检查代理 IP 是否被目标平台限制
3. 降低并发数，增加请求间隔
4. 切换备用代理重试
**预防**: 配置多代理轮换池，设置合理超时阈值（建议 30s）

### 1.4 元素定位失败
**现象**: "element not found" 或 "selector timeout"
**根因**: 目标页面结构变更（改版、AB测试）
**处理步骤**:
1. 查看采集截图确认页面状态
2. 对比新旧页面 HTML 差异
3. 更新选择器配置（使用更稳定的属性如 data-testid）
4. 增加备用选择器降级策略
**预防**: 配置页面结构变更监控，关键页面每日截图对比

### 1.5 数据库异常
**现象**: "connection refused" 或 "too many connections"
**根因**: MySQL 服务异常或连接池耗尽
**处理步骤**:
1. 登录服务器检查 MySQL 服务状态
2. 查看最大连接数配置 (show variables like 'max_connections')
3. 检查慢查询日志定位阻塞
4. 必要时重启 MySQL 或增加连接池大小
**预防**: 配置 DB 连接数监控告警，定期清理历史数据释放空间

## 2. 故障排查流程

### 2.1 快速定位
1. Admin → ETL执行记录 → 按时间筛选失败记录
2. 点击 trace_id 查看完整处理链路
3. Admin → SQL巡检 → 查询对应异常类型
4. Admin → 任务监控 → 点击展开店铺执行明细

### 2.2 恢复流程
1. 确认根因后修复配置或环境
2. Admin → 任务管理 → 重新执行失败任务
3. 验证 ODS 表数据是否正常入库
4. 更新告警状态为"已处理"

### 2.3 预防措施
1. 每日查看 AI 运营中心健康巡检评分
2. 关注 SQL 巡检中待处理告警
3. 定期检查任务成功率趋势
4. 维护告警知识库，积累历史案例

## 3. 平台接入流程

### 3.1 新增采集平台
1. 评估平台数据源可靠性（API / 页面）
2. 创建 ODS 表和 DW 表
3. 编写采集器（继承 BaseCollector）
4. 注册到 collector_registry
5. 配置文件夹路由（Admin → 路由配置）
6. Admin → 任务管理 → 新增任务
7. 测试采集 → 验证数据入库

### 3.2 新增监控 SQL
1. 编写数据质量校验 SQL
2. Admin → SQL巡检 → 新增监控 SQL
3. 设置负责人和 Cron 表达式
4. 手动执行一次验证结果

## 4. 性能优化

### 4.1 采集优化
- 接口优先于页面采集（规避 UI 解析）
- 使用 ThreadPoolExecutor 并发采集
- 配置增量采集时间窗口避免全量扫描
- 启用断点续采功能

### 4.2 数据库优化
- 按月分表减少单表数据量
- 核心查询字段建立索引
- 定期执行 ANALYZE TABLE 更新统计信息
- 历史数据归档至冷存储
