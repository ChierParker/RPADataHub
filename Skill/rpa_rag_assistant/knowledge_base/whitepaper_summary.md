# 基于RPA+工程化架构的规模化数据采集与运维实践白皮书 核心摘要

## 核心架构
四层解耦架构: 采集层(Playwright/RPA) → 处理层(ETL+数据校验) → 存储层(MySQL ODS/DW/DM) → 监控层(Admin+Grafana)

## 采集机制
双轨采集验证机制: RPA优先 + API兜底 + 双向验证，保障数据完整性
针对不同平台数据源可靠性分级匹配策略:
- L1 结构化API: 直接入库，轻量校验
- L2 可逆向接口: Ajax请求+数据量校验
- L3 页面UI采集: 双轨验证或降级机制
- Lx 非结构化源(邮件/PDF): 解析层+格式校验层

## 数据治理
ODS→DW→DM 三级分层建模，维表驱动质量校验
三层数据校验体系:
- L1 存在性: 店铺是否有数据、数据量是否骤降
- L2 一致性: 跨表交叉校验、ODS与DW对账
- L3 完整性: ASIN粒度字段缺失、格式合法性

## 运维体系
分级告警机制: P0紧急(实时推送+声音)/P1重要(实时推送)/P2一般(仅看板)
低频账号降频: 连续7天无数据→标记低频→每周二恢复检查
ETL联动阻断: 源数据校验→异常自动阻断入库→运维看板记录→关键异常推送

## 技术演进
从RPA到Playwright迁移策略:
- 成本: 年授权费节省10万+
- 稳定性: 可用性92%→99.5%+
- 并发: 单机器人串行→多线程并发(效率提升50%)
- AI扩展: 代码驱动架构可直接对接LangChain/AutoGen等AI框架
分三批迁移: 高价值高复杂度(6月)→中等复杂度(4月)→低频流程持续替换

## 分布式调度
Worker Agent + Redis List MQ (LPUSH/BRPOP) 实现实时任务推送
task_queue表保留为审计日志和降级兜底
支持多机并发采集，Admin一键启动

## AI 集成
- AI运营助手: DeepSeek Function Calling 自然语言查询数据库
- AI告警分析: 四步流程(上下文聚合→知识库检索→AI分析→结果落库)
- RAG知识库: 白皮书+运维手册+SQL规则 私有向量检索
- 3个Skill + 1个Agent: 健康巡检/任务日报/智能诊断 + RPAOps-Agent
